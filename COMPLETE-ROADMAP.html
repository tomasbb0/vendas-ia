1.1: Supabase - Base de Dados PostgreSQL

📋 O QUE É:

Base de dados para guardar: leads, mensagens geradas, clientes, histórico

💡 PORQUÊ SUPABASE:
  • Free tier generoso (500MB database, 50K users)
  • REST API automática (não precisas criar backend)
  • Auth built-in (login de clientes)
  • Real-time subscriptions
✅ PASSOS EXATOS:
  1. Abre supabase.com
  2. Click "Start your project" → Login com GitHub
  3. Cria projeto: Nome "vendasia", Region "Europe West"
  4. Espera 2min (provisioning)
  5. Vai a "Table Editor" → Cria 3 tabelas:
    • leads - name, company, title, linkedin_url, email, fit_score
    • messages - lead_id, message_text, sent_date, response_status
    • customers - name, email, plan, mrr, status
  6. Vai a "Settings" → "API" → Copia:
    • URL: https://xxx.supabase.co
    • anon key: eyJh...

3.3: Criar Primeiro Workflow - Lead Processor

📋 O QUE CRIAR:

Workflow que pega em CSV de leads → Gemini API → Guarda mensagens no Supabase

✅ PASSOS EXATOS:
  1. Abre o teu n8n: https://n8n-vendasia.up.railway.app
  2. Click "New Workflow"
  3. Adiciona nodes (drag & drop):
    1. Trigger: "Webhook" (para receber CSV)
    2. Parse: "Spreadsheet File" (ler CSV)
    3. Loop: "Split In Batches" (processar 1 por 1)
    4. AI: "HTTP Request" (chamar Gemini API)
    5. Save: "Supabase" node (guardar em DB)
  4. Configura cada node com as tuas API keys
  5. Testa com 1 lead
  6. Se funcionar → "Save" e "Activate"
